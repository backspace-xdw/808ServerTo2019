# JT808-2019服务器项目说明

## 📋 项目概述

本项目是基于 **JT/T 808-2019** 协议标准的车载终端通讯服务器,完整实现了2019版本的协议扩展,并向后兼容2013版本。

**创建时间**: 2025-11-23
**协议版本**: JT/T 808-2019 (兼容2013)
**开发语言**: C# (.NET 9.0)
**服务器端口**: 8809 (默认)
**当前状态**: 已完成高并发优化 + 实时位置存储 + 归档双写

---

## 🎯 核心特性

### 1. 协议版本自动识别
- 同时支持JT/T 808-2013和2019两个协议版本
- 根据消息体属性bit14自动识别版本
- 无需配置,自动适配客户端版本

### 2. 2019版本主要扩展

| 特性 | 2013版本 | 2019版本 | 说明 |
|------|---------|---------|------|
| 手机号长度 | 6字节 | 10字节 | 支持更长手机号 |
| 制造商ID | 5字节 | 11字节 | 更详细的制造商信息 |
| 终端型号 | 20字节 | 30字节 | 更长的型号描述 |
| 终端ID | 7字节 | 30字节 | 更长的终端标识 |
| 协议版本号 | 无 | 1字节 | 新增版本标识 |
| 鉴权信息 | 仅鉴权码 | +IMEI+软件版本 | 更丰富的终端信息 |
| 参数ID | 1字节 | 4字节 | 支持更多参数类型 |
| 位置附加信息 | 基础 | 扩展 | 新增多种附加信息类型 |

### 3. 新增消息类型

2019版本新增的消息ID:
- **0x0901** - 数据压缩上报
- **0x0A00** - 终端RSA公钥
- **0x8A00** - 平台RSA公钥
- **0x0801** - 多媒体数据上传
- **0x8801** - 摄像头立即拍摄命令
- **0x8802** - 存储多媒体数据检索
- 更多扩展消息...

### 4. 位置附加信息扩展

2019新增附加信息ID:
- **0x11** - 超速报警附加信息
- **0x12** - 进出区域/路线报警附加信息
- **0x13** - 路段行驶时间附加信息
- **0x25** - 扩展车辆信号状态位
- **0x2A** - IO状态位
- **0x2B** - 模拟量
- **0x30** - 无线通信网络信号强度
- **0x31** - GNSS定位卫星数卫星数

```csharp

public bool Is2019Version => (MessageBodyProperty & Protocol.MessageBodyProperty.VersionFlagMask) != 0;
```

### 2. 扩展字段解析

```csharp
// JT808Decoder.cs - 终端注册信息
if (is2019)

    ManufacturerId = ReadString(body, ref offset, 11);
    TerminalModel = ReadString(body, ref offset, 30);
    TerminalId = ReadString(body, ref offset, 30);

    TerminalModel = ReadString(body, ref offset, 20);

```

### 3. 位置附加信息

```csharp
// 附加信息格式: [ID(1)] [长度(1)] [内容(N)]
while (offset < body.Length)
{
    byte id = body[offset++];
    byte length = body[offset++];
    byte[] content = new byte[length];
    Array.Copy(body, offset, content, 0, length);
    offset += length;

    // 根据ID解析不同类型的附加信息
    ParseAdditionalInfo(id, content);
}
```

## 📈 性能指标 / 高并发优化

- **并发连接**: 10,000+ 终端 (默认硬上限 12000, 可调)
- **接收 SAEA 复用**: 每个 session 持有一份 `SocketAsyncEventArgs` + `ArrayPool<byte>` 缓冲区, 整个连接生命周期复用
- **零分配热路径**: `JT808MessageBuffer.Append(buffer, offset, count)` 直接消费接收 buffer, 不再 `new byte[]`
- **同步内联处理**: 接收回调内同线程处理消息, 不走 `Task.Run`, 避免 ThreadPool 抖动
- **per-session 发送锁**: 同 socket 并发 Send 串行化, 防止数据错乱
- **位置数据异步通道**: `Channel<string>` + 后台 worker, 同车短时多次上报去重合并, 调用方零等待
- **原子时间戳**: `LastActiveTicks` 用 `Interlocked` 读写, 替代 `DateTime.Now`
- **PeriodicTimer 清理**: 60s 周期清理超时会话, 不占工作线程
- **Socket 调优**: `NoDelay = true` (关 Nagle), `KeepAlive = true`, `SendTimeout = 5000`
- **连接数硬上限**: `MaxConcurrentConnections` 超限直接关闭新连接
- **版本开销**: 版本识别几乎无性能损耗
- **兼容模式**: 2013/2019 混合在线无影响

## 💾 实时位置数据存储

### 主存储

- **路径**: `LocationData/{12位手机号}.xml`
- **行为**: 始终保留每辆车**最新的一条**, 后到的覆盖前面的
- **写入方式**: 后台 worker 异步刷盘, 调用方零阻塞; 同手机号短时多次上报会合并去重
- **编码**: UTF-8 with BOM

### 归档双写 (可选)

- **配置**: `appsettings.json` 中 `LocationArchiveDirectory` 非空时启用
- **路径**: `LocationArchive/{yyyyMMdd}/{12位手机号}.xml`
- **行为**: 主存储写入时, 同时写一份到当天的日期文件夹下; 跨天自动建新文件夹
- **隔离**: 归档与主存储互不影响, 任一失败都不影响另一个

### XML 字段格式 (40 项, 与上游平台标准一致)

```
PhoneNumber / Time / ACCZT / DingweiZT / YunYinZT
Longitude / Latitude / GaoDu / Speed / direction / mileage / ilLevel
JinjiBJ / ChaoSuBJ / PiNaoJiaShi / WeiXianYJ / GNSSmokuaiGZ / GNSSTXWJGZ
GNSSTXDL / ZDZDYQY / ZDZDYDD / ZDLCDHXSQGZ / TTSMKGZ / SXTGZ / YSZICKMKGZ
CSYJ / PNJSYJ / DTLJJSCS / CSTCBJ / JCQYBJ / JCLXBJ / LDXSSJBZHGC / LXPLBJ
CLVSSGZ / CLYLYCBJ / CLBDBJ / CLFFDHBJ / CLFFWYBJ / PZYJ / CFYJ
```

报警字段对应 JT808-2019 报警标志位 0~14、18~30 (跳过保留位)。
direction 为弧度 (F2), mileage 单位 km (F6), 经纬度自然映射 (经度→`<Longitude>`, 纬度→`<Latitude>`)。
